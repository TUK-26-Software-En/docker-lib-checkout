## 2. 컨테이너 상세

### 2.1 library-postgres

| 항목 | 내용 |
|------|------|
| 이미지 | `postgres:16-alpine` |
| 컨테이너명 | `library-postgres` |
| 역할 | 도서·회원·대출 데이터 영구 저장 |
| 포트 | `${POSTGRES_PORT}:5432` (기본 5432:5432) |
| 환경 변수 | `POSTGRES_DB`, `POSTGRES_USER`, `POSTGRES_PASSWORD` |
| 볼륨 | `postgres_data:/var/lib/postgresql/data` (named volume) |
| Healthcheck | `pg_isready -U ${POSTGRES_USER} -d ${POSTGRES_DB}` (interval 10s, retries 5) |
| 네트워크 | `library-net` (bridge) |

---

### 2.2 library-checkout

| 항목 | 내용 |
|------|------|
| 빌드 컨텍스트 | `./library-checkout/library` |
| 기반 이미지 | `python:3.12-slim` |
| 컨테이너명 | `library-checkout` |
| 역할 | 도서 대출 관리 REST/HTML API + 헬스 체크 + 장애 주입 |
| 포트 | `${LIBRARY_PORT}:8000` (기본 8000:8000) |
| 의존성 | `postgres` (service_healthy 조건) |
| 시작 순서 | `entrypoint.sh` → pytest 전체 실행 → uvicorn 기동 |
| Healthcheck | `curl -sf -X POST http://localhost:8000/health` (interval 30s, start_period 90s) |
| 네트워크 | `library-net` |

**환경 변수**

| 변수 | 설명 |
|------|------|
| `DATABASE_URL` | PostgreSQL 연결 URL (docker-compose 내에서 조합) |
| `DB_TABLE_BOOKS` | books 테이블명 (기본: `books`) |
| `DB_TABLE_MEMBERS` | members 테이블명 (기본: `members`) |
| `DB_TABLE_LOANS` | loans 테이블명 (기본: `loans`) |
| `APP_TITLE` | FastAPI 앱 제목 |
| `APP_ENV` | `production` / `development` |
| `LOG_LEVEL` | uvicorn 로그 레벨 (기본: `info`) |

---

### 2.3 health-monitoring

| 항목 | 내용 |
|------|------|
| 빌드 컨텍스트 | `./health-monitoring` |
| 기반 이미지 | `python:3.12-slim` |
| 컨테이너명 | `health-monitoring` |
| 역할 | library-checkout 헬스 상태 폴링, MTBF·MTTR·MTTF·가용성 계산, 장애 주입 프록시 |
| 포트 | `${MONITOR_PORT}:8001` (기본 8001:8001) |
| 의존성 | `library-checkout` (service_healthy 조건) |
| 네트워크 | `library-net` |

**환경 변수**

| 변수 | 설명 |
|------|------|
| `LIBRARY_URL` | library-checkout 내부 URL (기본: `http://library-checkout:8000`) |
| `POLL_INTERVAL` | 헬스 체크 폴링 주기 (초, 기본: `30`) |
| `MAX_HISTORY` | 최대 이력 보관 수 (기본: `20`) |

## 3. 컨테이너 시작 순서

```
library-postgres  (healthy)
        ↓ depends_on
library-checkout  (healthy)   ← pytest 통과 후 uvicorn 기동
        ↓ depends_on
health-monitoring             ← asyncio 폴링 루프 시작
```

## 4. 강의 내용 적용 현황

| 강의 요소 | 구현 위치 |
|-----------|-----------|
| 3 Layer Architecture | `models/` → `repositories/` → `services/` → `routers/` |
| MVC 패턴 | `templates/` (View) · `routers/` (Controller) · `models/` (Model) |
| 캡슐화 / 정보 은닉 | Service가 Repository를 감싸고, Router는 Service만 호출 |
| 단위 테스트 | `tests/` — SQLite in-memory, 컨테이너 시작 시 자동 실행 |
| 컨테이너화 | Docker Compose 3-컨테이너 (DB·App·Monitor) |
| 헬스 체크 | POST /health (JSON), healthcheck 체인 |
| 장애 주입 | state.py 전역 플래그, /fault/inject|recover|status |
| 신뢰성 지표 | MTBF · MTTR · MTTF · Availability (health-monitoring) |
